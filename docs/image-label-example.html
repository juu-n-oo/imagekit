이미지 메타데이터 라벨 — 실제 빌드 예시

PRD 10절 "이미지 메타데이터 라벨(OCI 표준)"이 실제 빌드에서 어떻게 나타나는지, 한 건의 빌드 결과(sse-qa/dockerfile-build-test:0.1.0)로 입력 → Dockerfile → Harbor 이미지 라벨까지 따라가며 보여준다.

이미지aipub-harbor.cluster10.idc1.ten1010.io/sse-qa/dockerfile-build-test:0.1.0
다이제스트sha256:c6bc6b4dbd57f26258c467488decde5ea444e11b64a3a3f741c9bf476eeb8010
Base 이미지aipub-harbor.cluster10.idc1.ten1010.io/common/ubuntu:latest
빌드 시각2026-06-10 06:38 UTC · 빌더 Kaniko · amd64 / linux
Dockerfileid 14 · revision 20 · 작성자 joonadmin

1. 입력 — Dockerfile 에디터 "이미지 메타데이터"

사용자는 에디터의 접이식 "이미지 메타데이터" 섹션에서 표준 필드와 커스텀 라벨을 입력했다. (title은 Dockerfile 이름 dockerfile-build-test에서 자동)

필드입력값
Version0.1.0
Authorsjoonadmin (기본값 = 로그인 사용자)
LicenseMIT
URLhttps://github.com/ten1010-io/dockerizer
Documentationhttps://github.com/ten1010-io/dockerizer/docs
커스텀 라벨io.ten1010.aipub.foo = apple · io.ten1010.aipub.bar = banana

2. 생성된 Dockerfile

입력값은 Dockerfile 본문 최하단의 LABEL 블록으로 구워진다(에디터 ↔ 본문 양방향 반영). 즉 이 값들은 Dockerfile 안에 보존되어 재편집·재빌드 시 그대로 따라간다.

WORKDIR /workspace

CMD ["bash"]

LABEL org.opencontainers.image.title="dockerfile-build-test"
LABEL org.opencontainers.image.version="0.1.0"
LABEL org.opencontainers.image.authors="joonadmin"
LABEL org.opencontainers.image.licenses="MIT"
LABEL org.opencontainers.image.url="https://github.com/ten1010-io/dockerizer"
LABEL org.opencontainers.image.documentation="https://github.com/ten1010-io/dockerizer/docs"
LABEL io.ten1010.aipub.foo="apple"
LABEL io.ten1010.aipub.bar="banana"

3. 빌드 결과 — Harbor 이미지 라벨

빌드가 끝나면 Harbor 이미지 상세의 Overview → config → Labels에서 라벨을 확인할 수 있다. Dockerfile에 적힌 라벨에 더해, 빌드 시 자동으로 채워진 라벨(생성 시각·리비전·출처 등)이 함께 보인다.

4. 라벨 출처 매핑 ★

같은 화면에 섞여 보이는 16개 라벨이 각각 어디서 왔는지가 핵심이다. 둘로 나뉜다.

사용자 입력 에디터에서 입력 → Dockerfile LABEL로 기록(이미지·Dockerfile 양쪽에 보존) 자동 주입 빌드 시 Dockerizer가 채움 → 빌드 옵션으로 부착(Dockerfile에는 없음)

왜 나누나? 버전·작성자·라이선스처럼 사람이 정하고 잘 안 바뀌는 값은 Dockerfile에 남겨 재현·추적이 되게 하고, 생성 시각·리비전처럼 빌드마다 달라지는 값은 빌드 시 붙여 Dockerfile을 깔끔하게 유지한다. aipub.ten1010.io/* 출처 라벨 덕분에 이미지만 보고도 "id 14번 Dockerfile의 20번 리비전을 joonadmin이 빌드"했음을 역추적할 수 있다.

6. 읽는 법 요약

  • 이미지의 "무엇·왜·누가"를 알고 싶으면 → Harbor Overview의 Labels 또는 위 API의 config.Labels를 본다.
  • 사람이 정한 값(버전·작성자·라이선스·링크·커스텀)은 Dockerfile에도 남아 있으니, 재현·재빌드 시 Dockerfile만으로 동일하게 복원된다.
  • 출처(provenance) 추적은 aipub.ten1010.io/* 세 라벨로 한다(어느 Dockerfile·리비전·사용자).
  • 보안 스캔(취약점·SBOM)은 Harbor/Trivy가 별도로 채우며, Dockerizer는 라벨만 책임진다.
